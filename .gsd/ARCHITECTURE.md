## Components

### 1. Planning Module (`src/modules/plan`)
- **Purpose**: Core planning engine (PNL, SEC, SPL planning).
- **Key Files**: `SortMaster.jsx`, `PnlPlanning.jsx`.
- **Functionality**: Fetches IPO (Internal Production Order) data, applies sorting logic, and generates production plans.

### 2. Planning Settings (`src/modules/planSettings`)
- **Purpose**: Configuration for the planning engine.
- **Key Files**: `SortMasterSettings.jsx`, `ExclusionItems.jsx`, `ChildParts.jsx`.
- **Functionality**: Manages master data like child parts, exclusion lists, and sort prefixes.

### 3. Import Module (`src/modules/import`)
- **Purpose**: Data ingestion.
- **Key Files**: `IPOImport.jsx`.
- **Functionality**: Imports Excel files and maps them to the `ipo_master` table in Supabase.

### 4. Admin Panel (`src/modules/admin`)
- **Purpose**: System administration.
- **Key Files**: `AdminPanel.jsx`.
- **Functionality**: Manages companies, user profiles, and global system settings.

### 5. Layout & Core (`src/components/layout`, `src/contexts`)
- **Purpose**: Common UI and infrastructure.
- **Key Files**: `Navbar.jsx`, `AuthContext.jsx`.
- **Functionality**: Handles navigation, authentication state, and consistent branding.

## Data Flow

1. **Import**: User uploads Excel spreadsheet via `IPOImport.jsx` -> Data parsed and batch-inserted into `ipo_master`.
2. **Configuration**: Master data (child parts, exclusion items) set in `planSettings`.
3. **Planning**: `PnlPlanning.jsx` fetches active IPOs -> Filters based on master data -> User generates plan -> Results saved/exported (PDF/Excel).
4. **Dashboard**: `HomePage.jsx` consumes views like `v_ipo_summary` to show real-time stats.
